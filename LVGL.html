LVGL显示优化

LVGL时序

lcdc的时序为如下几个参数

  • HSYNC width：水平同步宽度设置，以LCD_CLK的像素时钟输出为单位
  • HBP(horizontal back porch period)：水平后沿周期设置，以LCD_CLK的像素时钟输出为单位
  • Active width：有效宽度设置，以LCD_CLK的像素时钟输出为单位。以800*480分辨率为例，Active width = 800
  • HFP(horizontal front porch period)：水平前沿周期设置，以LCD_CLK的像素时钟输出为单位
  • VSYNC width：垂直同步宽度设置，以LCD_CLK的像素时钟输出为单位
  • VBP(vertical back porch period)：垂直后沿周期设置，以LCD_CLK的像素时钟输出为单位
  • Active height：有效高度设置，以LCD_CLK的像素时钟输出为单位。以800*480分辨率为例，Active height = 480
  • VFP(vertical front porch period)：垂直前沿周期设置，以LCD_CLK的像素时钟输出为单位

LCDC刷新

正常情况下，LCD的刷新就是从左到右，从上到下进行逐个像素点刷新。但仅刷新有效的显示区是不够的，比如800480分辨率，我们不仅仅要刷800480这段有效区域，边界区也是要刷新的，即下图总宽度以内，有效宽度以外的区域。

  • 水平消隐就是LCD扫描一行结束到另一行开始的时间，即HSYNC宽度+ HBP + HFP

  • 垂直消隐就是LCD扫描最后一行结束到第一行开始的时间，即VSYNC宽度+ VBP + VFP

  • 刷新率 = LTDC输出时钟 /((Width + HSYNC_W + HBP + HFP) * (Height + VSYNC_W + VBP +VFP))

LVGL刷新方式

lvgl共有如下三种缓冲方式：

  • 单缓冲(Example 1: One buffer for partial rendering)
  • 非全屏双缓冲(Example 2: Two buffers for partial rendering)
  • 全屏双缓冲(Example 3: Two buffers screen sized buffer for double buffering.)

Display interface — LVGL documentation

如果只使用一个缓冲区，LVGL会将屏幕的内容绘制到该绘图缓冲区，并将其发送到显示器。这样，LVGL需要等到缓冲区的内容被发送到显示器之后，才能在其中绘制新的内容。

如果使用两个缓冲区，LVGL可以在一个缓冲区中绘制内容，同时另一个缓冲区的内容在后台发送到显示器。应该使用DMA或其他硬件来将数据传输到显示器，以便让MCU同时进行绘制。这样，显示的渲染和刷新就可以并行进行。

如果启用了full_refresh并且提供了两个屏幕大小的绘图缓冲区，则LVGL的显示处理工作方式类似于“传统”的双缓冲。这意味着在flush_cb中只需更改帧缓冲区的地址为提供的指针（color_p参数）。如果MCU具有LCD控制器外设而不是外部显示控制器（例如ILI9341或SSD1963），则应使用此配置。

垂直消隐+双缓冲

使用lvgl的前两种缓冲方式时，存在数据一致性问题，即cpu(dma)写入数据时，lcdc也在读取数据。area只刷新一半时，lcdc把全部数据读走，会导致撕裂的现象。

这个问题可以使用全屏双缓冲的方式来进行解决，lvgl渲染数据写入buffer1，lcdc显示读取buffer2。在垂直消隐期间，配置lcdc显示读取渲染好的buffer，lvgl使用另外一片buffer进行渲染。

具体思路如下(裸机)：

  • 配置lcdc在垂直消隐期间产生中断(根据lcdc ip不同进行配置)
  • 在中断处理程序中，设置全局变量标记中断产生
  • 修改disp_flush函数

LVGL模拟器使用（GUI-Guider）

  • 下载GUI-Guider
  • 将GUI-Guider安装目录下 environment\mingw\bin 添加到环境变量中
  • 将GUI-Guider安装目录下 environment\tools\unixTools 添加到环境变量中
  • 在GUI-Guider生成的代码目录下的 lvgl-simulator目录下，make 即可编译代码
  • make run启动模拟器
